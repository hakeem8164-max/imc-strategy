# منصة المساجد المتكاملة لإدارة الأداء

منصة ويب لإدارة مؤشرات أداء **شركة المساجد المتكاملة** — من إدخال القياسات إلى متابعتها عبر
لوحة معلومات تفاعلية، مع نظام مستخدمين وصلاحيات والهوية البصرية للشركة.

![الهوية البصرية](public/mushar-logo.png)

## المزايا

- **لوحة معلومات (Dashboard):** نظرة شاملة على المؤشرات، نسب الإنجاز مقابل المستهدف،
  وتوزيع المؤشرات حسب الأبعاد الاستراتيجية مع رسوم بيانية.
- **إدارة المؤشرات والإدخال:** تصفية وبحث في المؤشرات (32 مؤشرًا) حسب البعد أو المالك،
  وإدخال القياسات الدورية لكل مؤشر مع سجل تاريخي ومخطط تطوّر.
- **الأبعاد الاستراتيجية:** 8 أبعاد (مالي، العملاء والموردين، تطوير الأعمال، الشراكات،
  العمليات، الموارد البشرية، البنية التقنية، الحوكمة) بألوان هوية الشركة.
- **المستخدمون والصلاحيات (RBAC):**
  - **مدير النظام (admin):** صلاحية كاملة على كل المؤشرات والمستخدمين.
  - **مالك المؤشرات (owner):** يُدخل القياسات للمؤشرات المرتبطة بمنصبه فقط.
  - **مشاهد (viewer):** اطّلاع دون تعديل.
  - الحماية مفروضة على مستوى قاعدة البيانات عبر **Row Level Security (RLS)**.
- **واجهة عربية كاملة (RTL)** بخط Cairo وهوية الشركة اللونية.

## التقنيات

- **Next.js 14** (App Router) + **TypeScript** + **Tailwind CSS**
- **Supabase** (PostgreSQL + Auth + RLS)
- **Recharts** للرسوم البيانية

## الإعداد المحلي

1. ثبّت الحزم:
   ```bash
   npm install
   ```
2. انسخ ملف البيئة واملأ القيم من لوحة تحكم Supabase:
   ```bash
   cp .env.example .env.local
   ```
   ```
   NEXT_PUBLIC_SUPABASE_URL=https://YOUR_PROJECT_REF.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=YOUR_PUBLISHABLE_KEY
   ```
3. شغّل بيئة التطوير:
   ```bash
   npm run dev
   ```
   ثم افتح http://localhost:3000

## قاعدة البيانات

ملفات الترحيل في `supabase/migrations`:

| الملف | الوصف |
|------|-------|
| `0001_schema.sql` | الجداول: `dimensions`, `profiles`, `kpis`, `kpi_entries` + إنشاء الملف تلقائيًا عند التسجيل |
| `0002_rls.sql` | سياسات الأمان على مستوى الصف ودوال `is_admin` و`can_edit_kpi` |
| `0003_seed.sql` | البيانات الأولية: 8 أبعاد و32 مؤشرًا من العرض التقديمي |
| `0004_lock_down_functions.sql` | تقييد تنفيذ الدوال الداخلية (معالجة تنبيهات الأمان) |

## النشر على Vercel

1. اربط المستودع بمشروع Vercel.
2. أضِف متغيرات البيئة `NEXT_PUBLIC_SUPABASE_URL` و`NEXT_PUBLIC_SUPABASE_ANON_KEY`.
3. أضِف نطاق Vercel إلى **Supabase → Authentication → URL Configuration**.

